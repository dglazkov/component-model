Component Model DOM Interfaces

The component model extends the Element as follows:

[Supplemental]
interface Element {
    readonly attribute ShadowRoot shadow;
    ShadowRoot removeShadow() raises (DOMException);
    attribute String pseudo;
}

The shadow attribute returns the current shadow root, hosted by this element. It returns null if the element is not currently hosting a shadow DOM subtree or its shadow DOM subtree is not accessible.

The removeShadow method removes the current shadow root hosted by this element. If the element does not host a shadow root or its shadow DOM subtree is not accessible a HIERARCHY_REQUEST_ERR DOM Exception is raised.

The pseudo attribute allows setting a CSS pseudo-element value on an element and roughly corresponds to functionality of the XBL2 pseudo attribute.

An element's shadow root is represented as a ShadowRoot object: 
interface ShadowRoot : TreeScope {
    attribute bool applyAuthorSheets;
    readonly attribute Element shadowHost;
    static ShadowRoot create(in Element element) raises (DOMException);
}

The applyAuthorSheets attribute indicates whether or not rules in author style sheets associated with the element's document apply to the shadow DOM subtree. It is an equivalent of the XBL2 apply-author-sheets attribute.

The shadowHost attribute refers back to the element that is hosting this shadow root.

The static create method attempts to create a shadow root, hosted at the element (supplied as the argument) and returns a newly instance of ShadowRoot object, if successful. If the element already hosts a shadow root, a HIERARCHY_REQUEST_ERR DOM Exception is raised. If creation of the shadow root is not possible on this element, null is returned.

A ShadowRoot is a TreeScope, which represents a DOM tree scope. 

interface TreeScope : Node {
    readonly TreeScope parentTreeScope;
    Element getElementById(in DOMString elementId);
}

The parentTreeScope attribute returns the tree scope that contains this tree scope.

The getElementById returns first element, in tree order, within the tree scope's tree, whose ID is elementId. It is essentially document.getElementById from DOM Core, but scoped inside of the tree scope.

Subsequently, a Document is also a TreeScope.
